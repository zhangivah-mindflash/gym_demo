# MVP 范围定义

## 产品目标

当前阶段的目标不是做一个完整健身平台，而是做一个 `可信、可展示、可演示` 的智能健身助手 Demo。

产品定位已经明确为：

- 面向健身房/工作室会员体系
- 平台用户主要是机构内会员
- 平台教练是机构内真人教练
- AI 的角色是放大教练服务效率，而不是替代真人教练

MVP 需要证明四件事：

1. 系统可以采集用户信息并识别其训练目标
2. 系统可以输出结构化训练计划并保存到个人页面
3. 系统可以对动作指导提供带引用、可解释的建议
4. 系统可以根据训练记录做一次复盘并更新计划

## 用户角色

### 1. 会员用户

核心任务：

- 注册/登录
- 填写基础身体数据
- 选择目标与约束
- 查看训练计划
- 查询动作指导
- 记录训练完成情况
- 查看复盘结果和更新后的计划
- 必要时接收真人教练补充建议

### 2. 管理员

核心任务：

- 管理 RAG 知识库
- 配置模型与提示词
- 查看用户数据
- 查看 Agent 对话
- 查看计划与复盘记录

### 3. 教练/运营人员

核心任务：

- 查看会员画像与计划
- 查看 AI 生成结果与复盘记录
- 必要时补充人工建议
- 协助维护动作规范与安全知识

## MVP 功能范围

### 一、计划生成

输入：

- 年龄、性别、身高、体重
- 训练目标：增肌 / 减脂 / 塑形 / 提高表现
- 每周可训练天数
- 每次训练时长
- 器械条件：居家 / 健身房 / 有限器械
- 训练经验：新手 / 进阶 / 高级
- 伤病史或禁忌动作
- 饮食偏好（先做轻量字段）
- 所属门店 / 教练归属（机构场景字段）

输出：

- 训练周期目标
- 每周训练结构
- 每日动作计划
- RPE / 休息建议
- 替代动作建议
- 简化饮食建议
- 人工教练补充位

展示形式：

- 计划总览卡片
- 周计划表格
- 日计划详情抽屉或详情页

### 二、动作指导

能力范围：

- 动作要点
- 常见错误
- 热身建议
- 放松建议
- RPE 与休息建议
- 风险提醒
- 替代动作推荐

MVP 限制：

- 视频资源先允许为空链接或占位视频
- 高风险建议必须带知识库引用
- 对疼痛、眩晕、既往伤病等信号触发更严格的安全提示

### 三、复盘迭代

输入：

- 是否完成训练
- 实际完成组数/次数
- 主观疲劳
- 训练感受
- 体重变化
- 围度变化（可选）

输出：

- 本周完成率
- 疲劳与恢复判断
- 计划偏差分析
- 下周调整建议
- 新版本计划

### 四、管理员后台

MVP 仅保留最必要能力：

- 知识库文档上传/启用/停用
- 模型配置项展示与保存
- Prompt 模板展示与保存
- 用户列表与基本画像
- 对话记录查看
- 计划版本查看
- 教练视角查看会员计划

## 暂不纳入 MVP

- 广告、商城、支付、社交
- 真实穿戴设备接入
- 多语言
- 多租户
- 教练排班与课程体系
- 复杂营养分析
- 图片/视频动作识别
- 医疗建议与诊断能力
- 对外开放的陌生用户注册体系

## MVP 成功标准

如果要把这版作为演示样板，我建议成功标准定成下面这样：

1. 用户能在 3 分钟内完成画像录入并生成首版计划
2. 计划结果能稳定渲染为结构化卡片/表格
3. 动作指导回答包含引用来源和风险提示
4. 用户完成一次训练记录后，系统能产出复盘与计划更新
5. 管理员能通过后台切换知识库或模型配置
6. 能清楚体现 AI 结果可被真人教练查看和补充

## 建议的演示路径

为了保证 Demo 演示顺畅，建议主讲路径固定为：

1. 会员注册并填写画像
2. 用户选择“8 周减脂，每周 4 练，每次 45 分钟”
3. 系统生成结构化计划并存入个人主页
4. 用户点击某个动作查看动作指导与引用
5. 用户提交一次训练记录
6. 系统生成复盘结果并给出下周调整
7. 教练/管理员进入后台查看知识库、对话记录与会员计划

## 我建议的最小数据字段

### 用户画像字段

- `user_id`
- `age`
- `sex`
- `height_cm`
- `weight_kg`
- `goal`
- `training_days_per_week`
- `session_minutes`
- `equipment_access`
- `training_level`
- `injury_history`
- `diet_preference`
- `wearable_permission_status`
- `gym_location_id`
- `coach_id`

### 计划字段

- `plan_id`
- `user_id`
- `plan_version`
- `goal_summary`
- `cycle_weeks`
- `weekly_structure`
- `daily_workouts`
- `progression_rules`
- `recovery_notes`
- `created_at`

### 动作指导字段

- `exercise_name`
- `guidance_summary`
- `warmup`
- `cooldown`
- `rpe_rule`
- `rest_rule`
- `risk_alerts`
- `alternative_exercises`
- `citations`

### 复盘字段

- `review_id`
- `user_id`
- `plan_id`
- `completion_rate`
- `subjective_fatigue`
- `weight_trend`
- `notes`
- `adjustment_summary`
- `next_plan_version`
