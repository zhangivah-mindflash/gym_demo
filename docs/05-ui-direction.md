# 前端设计方向

## 设计目标

用户侧前端需要同时传达四种感受：

- 可靠
- 专业
- 健康
- 年轻

这不是一个花哨的 AI 聊天页，而是一个面向健身房/工作室会员的数字化训练服务入口。  
视觉上需要比普通 SaaS 更有张力，但不能走廉价、炫技、赛博风。

## 用户侧视觉定位

建议采用以下整体方向：

- 气质：现代训练品牌感，而不是科技概念感
- 颜色：清爽、克制、偏运动科学，不要紫色科技风
- 排版：有节奏感和力量感，强调训练卡片与数据模块
- 组件：重点突出计划、进度、风险提醒、引用来源

## 建议的视觉关键词

- clean performance
- athletic editorial
- coached precision
- healthy confidence

## 建议的色彩方向

建议使用 `浅底 + 强对比强调色` 的体系。

可考虑的主色方向：

- 主背景：暖白、浅灰、轻微米色
- 主文本：深墨绿或深石墨色
- 强调色：训练感较强的深绿、青绿、钴蓝其一
- 警示色：克制的橙红
- 成功色：自然绿色

避免：

- 大面积纯黑
- 紫色科技风
- 高饱和渐变堆叠
- 过多玻璃拟态

## 页面风格建议

### 1. 首页 / 登录后首页

需要体现：

- 今日训练状态
- 本周计划完成度
- 当前目标
- 我的教练
- AI 最新建议

首页不应该像消息列表，而应该像一个 `训练驾驶舱`。

### 2. 我的计划页

重点是结构清晰：

- 周计划总览
- 每日训练卡片
- 当前版本与上次更新时间
- 教练备注
- AI 调整原因

计划页应该更像训练台账，而不是聊天记录。

### 3. 动作指导页

重点是可信与安全：

- 动作名称与目标肌群
- 动作要点
- 常见错误
- 热身与放松
- 风险提醒
- 引用来源

引用信息不能藏得太深，否则“可信”这件事展示不出来。

### 4. 复盘页

重点是让用户看到“系统真的在跟进我”：

- 完成率
- 疲劳反馈
- 本周变化
- 调整逻辑
- 下一版计划摘要

## 信息架构建议

用户端建议主导航为：

- 首页
- 我的计划
- 动作指导
- 训练记录
- 复盘

次级区域可展示：

- 我的教练
- 设备接入
- 个人资料

## 交互原则

### 1. 任务优先

优先让用户完成任务，不要默认把界面做成大聊天框。

### 2. 结构化优先

计划、指导、复盘都应该优先卡片化、表格化、模块化展示。

### 3. 风险信息显式化

涉及禁忌、疼痛、停止训练提示时，必须清楚、固定、醒目。

### 4. 教练关系可见

用户要感受到不是在对着一个匿名机器人，而是在使用“AI + 真人教练”的服务。

## 后台设计方向

后台不需要过度包装品牌感，但要显得稳、清楚、专业。

建议风格：

- 信息密度中高
- 强调配置、状态、版本、引用来源
- 重点突出知识库、模型、Prompt、会员数据、对话记录

## 开发前的界面结论

如果后续开始做页面，我建议优先做这 5 个关键界面：

1. 会员首页仪表盘
2. 计划生成与计划详情页
3. 动作指导页
4. 复盘页
5. 管理员/教练后台首页

这 5 个页面足够支撑第一版完整演示，也最能体现产品差异。
